PairTwo
*****************************************************

CSV: une grosse nouveauté: Il est maintenant possible d'importer des fichiers d'inscriptions au format csv (issu de Excel). Cette importation ce fait à partir du menu 'Tournoi, Import' ou avec le 'raccourci clavier' Ctrl I. Ce fichier peut provenir de l'application Excel ou être composé à partir d'un éditeur de texte. Chacun des champs sera terminé par un 'point-virgule'. Le dernier 'point-virgule' est facultatif.
Le fichier contiendra dans l'ordre maximum 10 champs:

    Matricule Belge; Champ facultatif. S'il est donné, le reste des renseignements proviendra des fichiers PLAYER.DBF et FIDE.DBF, les autres champs sont ignorés.
    Matricule Fide; Champ facultatif. S'il est donné, le reste des renseignements proviendra des fichiers PLAYER.DBF et FIDE.DBF, les autres champs sont ignorés.
    S'il n'y a pas de matricule belge ni FIDE, alors les champs suivant doivent être compétés
    Nom;
    Prénom;
    Elo National; ZERO ou absent == NON CLASSE
    Date de Naissance; format YYYYMMDD
    Sexe; F ou M (par défault=M)
    Nationalité; en 3 lettres. BEL par défault
    N° de Club;
    Catégoorie

Example:
83224 (matricule national, MOI)
83224; (matricule national avec ou sans virgules)
;200018; (pas de matricule national; matricule FIDE, Luc Winand)
;;Marchal;Georges;1534;19440810;M;BEL;601; (toujours MOI avec Nom;prénom)
;;Marchal Georges;;1534;19440810;;;601; (et encore MOI avec valeurs par default et Nom Prénom)
;;Nouveau;joueur;;19780101;;;601;3 (un nouveau joueur NC avec catégorie 3)
;;Autre;Nouveau;1400;;;;601; (un nouveau avec un ELO estimé de 1400)

SWAR
*****************************************************

SWAR permet d'importer un fichier .csv (créé à partir de excel ou de open-office) afin d'importer des joueurs dans un tournoi. Il permet aussi d'intégrer les données du tournoi, les catégories à attribuer et tous les joueurs.
Jusque maintenant, ce fichier ne servait qu'à importer les joueurs dans un tournoi dont la description a déjà été faite. Mais il m'est apparu que ce serait aussi facile d'y intégrer les autres données.
Voici la description de ce fichier .csv et comment l'utiliser :
En premier lieu, une entrée a été ajoutée dans le menu :

    Les données sont précédées d'un Tag pour les distinguer, [TOURNOI] [CATEGORIE] [JOUEUR].
    Ces Tags peuvent aussi être les valeurs qui se trouvent dans le fichier Swar.Langue.xx.ini.
    Tournoi en Francais(_TOURNOI=Tournoi)
    Toernooi en Neerlandais (_TOURNOI=Toernooi)
    Tournament en Anglais ( _TOURNOI=Tournament)
    Même chose pour les valeurs reprisent dans _CATEGORIE _JOUEUR
    Donc [Tournoi] en français (_TOURNOI = Tournoi) et [Toernooi] en neerlandais. (voir Swar.Lang.nl.ini : _TOURNOI = Toernooi
    Après le Tag, viennent les données utiles. Le Tag doit se trouver dans la première colonne et rien d'autre ne doit se trouver sur la même ligne.
    Des lignes blanches peuvent être incorporées au fichier pour plus de lisibilité.

Description des Tags
[TOURNOI]
Description du tournoi sur 2 colonnes. La première colonne contient un chiffre désignant le type de donnée, la seconde colonne la description. Les données sont facultatives.

    Nom du tournoi
    Organisateur
    Lieu
    Date Début (jj-mm-aa, jj-mm-aaaa ou aaaa-mm-jj avec séparateur '-' ou '/')
    Date fin (mêmes restrictions que la date de début)
    Swiss est le seul type autorisé actuellemente.
    Std , Rapid, Blitz sont les 3 mots autorisés.
    Nombre de rondes

Haut de page

[CATEGORIE]
Description de la catégorie sur 2 colonnes. Cette rubrique est autorisée uniquement pour les tournoi Swiss. La première colonne contient un chiffre désignant le n° de la catégorie, la seconde colonne contient soit un ELO maximum, soit un AGE maximum. Pour désigner un ELO il faut juste mettre l'ELO maximum qui est toujours supérieur à 1000 alors que pour une catégorie d'age le nombre sera inférieur à 100 (14, 16, 20 ...) ceci afin de distinguer le type de catégorie. Un nombre Maximum de 15 catégories est utilisé par SWAR.
Exemple de catégorie ELO

    2000 (correspond au joueurs de moins de 2000 ELO
    1800
    1600
    1400

Exemple de catégorie d'AGE

    20 (correspond au joueurs de moins de 20 ans
    16
    14
    12

Haut de page

[JOUEUR]
Description des joueurs. Chaque joueur se trouve sur une seule ligne, les données sont séparées par le séparateur donné dans les options de SWAR. La description est fixe. Si un matricule national est entré, les renseignements sont pris dans le fichier PLAYER.DBF. Si c'est un Id FIDE, ils sont lus dans le fichier FIDE.DBF. Si aucun matricule n'est renseigné il faut alors au minimum entrer un Nom.
Description de la ligne Joueur

    Matricule (< 100000) ou Id Fide (> 100000)
    Nom et prénom
    Sexe (M F W V)
    Club
    Date Naissance (jj-mm-aa, jj-mm-aaaa ou aaaa-mm-jj. Le séparateur peut être au choix '-' ou '/')
    Elo
    Titre ("GM" "IM" "FM" "CM" "WGM" "WIM" "WFM" "WCM")
    Nationalité
    N° ou Nom de la catégorie (2 ou 1800 ou 16)
